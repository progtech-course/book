== TTY

Аббревиатура *tty* расшифровывается как _teletype_ -- физическое устройство, некогда использовавшееся для отправки и получения сообщений. В Linux tty-устройства отражают _виртуальные терминалы_, через которые пользователь взаимодействует с системой. Терминал в данном случае следует воспринимать не как окно, в которое вводят команды, а как некоторое устройство, через которое идет взаимодействие с пользователем.

Под терминалы создаются файлы `/dev/ttyN`. Чтобы сменить виртуальный терминал, используется команда `chvt TERMINAL_ID` или комбинация клавиш kbd:[Ctrl,Alt,F#] (`#` здесь отражает индекс терминала). Файл `/dev/tty` указывает на текущий контролирующий терминал.

Графическая оболочка открывается поверх некоторого виртуального терминала и транслирует действия с графическим интерфейсом в команды для виртуального терминала. Обычно для этого используются устройства `/dev/tty2` или `/dev/tty7`.

Чтобы узнать, с каким терминалом вы сейчас взаимодействуете, можно выполнить команду `tty`.

== Console

Файл `/dev/console` же отражает за системную консоль, которая используется при стартапе системы. Данный терминал управляется напрямую ядром и назначается init-процессу при загрузке системы.

== PTS

Графическое окно, которое открывается в операционной системе, на самом деле является запущенным *эмулятором*. Примерами такого могут быть `xterm`, `xterm-kitty`.

[cols="2*", frame="none", grid="none"]
|===
|image:logos/kitty.svg[kitty,100,role=text-left]
|image:logos/xterm.png[kitty,100,role=text-center]
|===

При запуске эмулятора создается пара из master и slave терминалов с помощью устройства `/dev/ptmx` -- pseudoterminal multiplexor device. Для второго создается устройство в папке `/dev/pts`, через которое происходит взаимодействие при работе в окне эмулятора.

NOTE: Под `/dev/pts` монтируется отдельная файловая система: _devpts_.
